# 1. Ôn tập

Buổi trước chúng ta đã học:

```text-x-trilium-auto
class Student:

    def __init__(self, name, age):
        self.name = name
        self.age = age

    def introduce(self):
        print(f"Tôi là {self.name}")
```

Trong đó:

- `Student` là **Class**
- `student = Student(...)` là **Object**
- `name`, `age` là **Instance Attribute**
- `introduce()` là **Method**

Hôm nay chúng ta sẽ đi sâu hơn vào cách Python quản lý dữ liệu trong Class.

# 2. Instance Attribute (Thuộc tính của đối tượng)

Ví dụ:

```text-x-trilium-auto
class Student:

    def __init__(self, name):
        self.name = name
```

Tạo hai đối tượng:

```text-x-trilium-auto
s1 = Student("An")
s2 = Student("Bình")
```

Kiểm tra:

```text-x-trilium-auto
print(s1.name)
print(s2.name)
```

Kết quả:

```text-x-trilium-auto
An
Bình
```

Mỗi đối tượng có một bản sao dữ liệu riêng.

Có thể hình dung:

```text-x-trilium-auto
Student
    │
 ┌──┴──┐
 │     │
s1     s2
 │      │
An    Bình
```

---

# 3. Class Attribute (Thuộc tính của lớp)

Đôi khi có dữ liệu dùng chung cho mọi đối tượng.

Ví dụ:

```text-x-trilium-auto
class Student:

    school = "ABC High School"

    def __init__(self, name):
        self.name = name
```

Sử dụng:

```text-x-trilium-auto
print(Student.school)

s1 = Student("An")

print(s1.school)
```

Kết quả:

```text-x-trilium-auto
ABC High School
ABC High School
```

`school` chỉ tồn tại **một lần** trong Class và được mọi đối tượng cùng sử dụng.

---

# 4. Sự khác nhau

```text-x-trilium-auto
class Student:

    school = "ABC"

    def __init__(self, name):
        self.name = name
```

| Instance Attribute | Class Attribute |
| --- | --- |
| `self.name` | `school` |
| Mỗi object có riêng | Dùng chung |
| Lưu trong object | Lưu trong class |

# 6. Điều gì xảy ra khi ghi đè?

```text-x-trilium-auto
s1.school = "Private School"
```

Lúc này:

```text-x-trilium-auto
print(s1.school)
```

```text-x-trilium-auto
Private School
```

Nhưng:

```text-x-trilium-auto
print(s2.school)
```

```text-x-trilium-auto
XYZ
```

Vì Python tạo một **Instance Attribute mới**, che khuất (shadow) Class Attribute.

Đây là lỗi rất phổ biến khi mới học OOP.

# 7. `@classmethod`

Giả sử muốn đếm số sinh viên.

```text-x-trilium-auto
class Student:

    total = 0

    def __init__(self):
        Student.total += 1
```

Có thể:

```text-x-trilium-auto
print(Student.total)
```

Nhưng cách chuyên nghiệp hơn:

```text-x-trilium-auto
class Student:

    total = 0

    def __init__(self):
        Student.total += 1

    @classmethod
    def get_total(cls):
        return cls.total
```

Sử dụng:

```text-x-trilium-auto
print(Student.get_total())
```

`cls` là chính Class đang được gọi.

---

# 8. `@staticmethod`

Có những hàm liên quan đến Class nhưng không dùng `self` hay `cls`.

Ví dụ:

```text-x-trilium-auto
class Math:

    @staticmethod
    def add(a, b):
        return a + b
```

Gọi:

```text-x-trilium-auto
Math.add(10, 20)
```

Khi nào dùng?

Ví dụ:

```text-x-trilium-auto
class Student:

    @staticmethod
    def is_valid_age(age):
        return age >= 18
```

Đây là hàm tiện ích, không cần truy cập dữ liệu của lớp hay đối tượng.

---

# 9. Magic Methods (Dunder Methods)

Magic Methods là các phương thức có dạng:

```text-x-trilium-auto
__tên__
```

Ví dụ:

```text-x-trilium-auto
__init__

__str__

__repr__

__len__

__eq__

__add__
```

Python tự động gọi chúng trong nhiều tình huống.

---

# 10. `__str__`

Ví dụ:

```text-x-trilium-auto
class Student:

    def __init__(self, name):
        self.name = name
```

In:

```text-x-trilium-auto
s = Student("An")

print(s)
```

Kết quả mặc định:

```text-x-trilium-auto
<__main__.Student object at 0x...>
```

Không thân thiện.

---

Định nghĩa:

```text-x-trilium-auto
class Student:

    def __init__(self, name):
        self.name = name

    def __str__(self):
        return f"Student({self.name})"
```

Kết quả:

```text-x-trilium-auto
Student(An)
```

---

# 11. `__repr__`

`__repr__` dành cho lập trình viên.

```text-x-trilium-auto
class Student:

    def __repr__(self):
        return f"Student(name='{self.name}')"
```

Thông thường:

- `__str__`: đẹp, dễ đọc.
- `__repr__`: chi tiết, phục vụ debug.

Nếu không có `__str__`, Python sẽ dùng `__repr__`.

---

# 12. `__len__`

Ví dụ:

```text-x-trilium-auto
class ShoppingCart:

    def __init__(self):
        self.items = []

    def __len__(self):
        return len(self.items)
```

Sử dụng:

```text-x-trilium-auto
cart = ShoppingCart()

print(len(cart))
```

Python tự gọi:

```text-x-trilium-auto
cart.__len__()
```

---

# 13. `__eq__`

So sánh hai đối tượng.

```text-x-trilium-auto
class Student:

    def __init__(self, student_id):
        self.student_id = student_id

    def __eq__(self, other):
        return self.student_id == other.student_id
```

Ví dụ:

```text-x-trilium-auto
a = Student("SV001")
b = Student("SV001")

print(a == b)
```

Kết quả:

```text-x-trilium-auto
True
```

---

# 14. `__add__`

Cho phép dùng toán tử `+`.

```text-x-trilium-auto
class Money:

    def __init__(self, amount):
        self.amount = amount

    def __add__(self, other):
        return Money(self.amount + other.amount)

    def __str__(self):
        return f"{self.amount:,} VND"
```

Ví dụ:

```text-x-trilium-auto
m1 = Money(100000)
m2 = Money(250000)

print(m1 + m2)
```

Kết quả:

```text-x-trilium-auto
350,000 VND
```

---

# 15. `@dataclass`

Nhiều Class chỉ dùng để lưu dữ liệu.

Ví dụ:

```text-x-trilium-auto
class Product:

    def __init__(self, name, price):
        self.name = name
        self.price = price
```

Python hiện đại hỗ trợ:

```text-x-trilium-auto
from dataclasses import dataclass

@dataclass
class Product:
    name: str
    price: float
```

Python sẽ tự tạo:

- `__init__`
- `__repr__`
- `__eq__`

Giúp giảm rất nhiều mã lặp.

---

# 16. Giá trị mặc định

```text-x-trilium-auto
from dataclasses import dataclass

@dataclass
class User:
    username: str
    active: bool = True
```

Sử dụng:

```text-x-trilium-auto
u = User("garden")

print(u)
```

Kết quả:

```text-x-trilium-auto
User(username='garden', active=True)
```

# 18. Những lỗi phổ biến

## Lỗi 1

Nhầm giữa:

```text-x-trilium-auto
self.total
```

và

```text-x-trilium-auto
Student.total
```

Nếu `total` là dữ liệu dùng chung, hãy truy cập qua lớp hoặc `cls`.

---

## Lỗi 2

Ghi đè Class Attribute:

```text-x-trilium-auto
student.school = "XYZ"
```

Việc này tạo Instance Attribute mới thay vì thay đổi Class Attribute.

---

## Lỗi 3

Quên `return` trong:

```text-x-trilium-auto

```

`__str__` phải luôn trả về một chuỗi (`str`).

---

## Lỗi 4

So sánh Object mà không định nghĩa:

```text-x-trilium-auto

```

Khi đó Python mặc định so sánh địa chỉ bộ nhớ, không phải nội dung.

# Góc lập trình viên chuyên nghiệp

Nếu đọc mã nguồn của các thư viện lớn như:

- Django
- FastAPI
- SQLAlchemy
- Requests
- Pydantic

bạn sẽ bắt gặp rất nhiều:

- Magic Methods (`__iter__`, `__getitem__`, `__enter__`, `__exit__`, ...).
- `@classmethod` để tạo các constructor thay thế (alternative constructors).
- `@dataclass` hoặc các thư viện tương tự như **Pydantic** để mô tả dữ liệu.

Việc hiểu rõ các cơ chế này sẽ giúp bạn không chỉ **sử dụng** thư viện, mà còn **đọc và thiết kế** mã nguồn theo phong cách Pythonic.

# Chuẩn bị cho Buổi 18

Ở **Buổi 18**, chúng ta sẽ học một trong bốn trụ cột quan trọng nhất của OOP:

# **Đóng gói (Encapsulation)**

Nội dung sẽ bao gồm:

- Public, Protected và Private.
- Quy ước `_name` và `__name`.
- Name Mangling hoạt động như thế nào.
- `@property` để kiểm soát truy cập dữ liệu.
- Getter và Setter theo phong cách Python.
- Thiết kế lớp an toàn, tránh dữ liệu bị sửa sai.
- Ứng dụng trong các hệ thống ngân hàng, thương mại điện tử và quản lý người dùng.

Sau buổi này, bạn sẽ biết cách xây dựng các lớp **an toàn, dễ bảo trì và đúng chuẩn thiết kế phần mềm hiện đại**.
